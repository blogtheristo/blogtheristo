# Azure AI Components Chart

## Introduction
This chart provides an overview of various Azure AI components and their interrelationships, helping professionals understand how these technologies can work together in modern AI solutions.

## Components
- **AI Search**: A service that enables powerful search capabilities across various data sources.
- **AI Foundry**: A platform for building and deploying AI models.
- **Azure Machine Learning**: A cloud-based environment for building, training, and deploying machine learning models.
- **GenAI**: Generative AI tools for creating content and insights.
- **Agentic Workflows**: Automated workflows that leverage AI to enhance productivity.
- **Retrieval-Augmented Generation (RAG)**: A method that combines retrieval and generation for improved outcomes.
- **Multi-Party Computation (MPC)**: A technology that allows parties to jointly compute a function over their inputs while keeping those inputs private.
- **Continuous Integration/Continuous Deployment (CICD)**: A practice that automates the integration and deployment of code changes.
- **Terraform**: An infrastructure as code tool that allows users to define and provision data center infrastructure.
- **Bicep**: A domain-specific language (DSL) for deploying Azure resources.
- **Data Science**: The field that involves using scientific methods and algorithms to extract insights from data.

## Relationships
- AI Search and AI Foundry are often integrated to enhance search capabilities with AI models.
- Azure Machine Learning is central to developing models for GenAI and other AI services.
- Agentic Workflows utilize AI Search and Azure Machine Learning to automate processes.
- RAG techniques can enhance the capabilities of GenAI by providing relevant data for content generation.
- MPC can be utilized in scenarios where data privacy is crucial, and it may integrate with Azure Machine Learning for secure model training.
- CICD processes are essential for maintaining and deploying updates to AI models in Azure.
- Terraform and Bicep are used to automate the deployment of infrastructure needed for these components.
- Data Science underpins all these technologies by providing the methodologies and frameworks for analysis and model building.
